### <p> Проект 1: Анализ A/B Теста</p>
<p>Что нужно было сделать:<p>
<ol>
  <li>Задача №1</li>
  Проанализировать АБ-Тест, проведенный во всех городах.  
  <li>Задача №2.</li>
  Визуализировать результаты 
  <li>Задача №3.</li>
  Провести сегментацию
  <li>Задача №4.</li>
  Сделать выводы и сформулировать рекомендации для дальнейших запусков АБ Теста
</ol>

<p>Как решал: краткое описание решения (автореферат)<p>

1. **Импорт и анализ таблиц.**
- Импортировал данные в окружение Jupyter Notebook.
- Изучил все три таблицы и проверил их на наличие пустых значений.
- Исключил из таблиц все строки, в которых есть нулловые значения.
- Исследовал количество торговых точек в каждом городе - построил группировку по количеству в каждом городе и визуализировал с помощью гистограммы.
2. **Объединение таблиц.**
- Построил агрегацию таблицы с платежами, где вычислил сумму платежей на каждого клиента.
- Соединил сгруппированную таблицу с платежами с клиентской таблицей. Убедился, что в таблице остались все клиенты из клиентской таблицы.
- Заполнил нулями суммы покупок тех клиентов, которые их не совершали.
- С помощью словаря регионов подтянул к каждой торговой точке город, в котором она находится.
- Создал поле флаг платежа, который принимает значения 0 или 1 в зависимости от того, заплатил клиент или нет.
3. **Автоматизация статистических вычислений.**
- Создал функцию ***test_calc***, которая будет вычислять значение t-критерия (критерия Стьюдента) и *p_value* для сравнения средних и с помощью функции *print* выводить сообщение о том, существует ли разница между средними (на основании *p_value*).
*Аргументы функции:* *r1* (первая выборка), *r2* (вторая выборка), *alpha* (уровень значимости - имеет значение по умолчанию 5%).
- Создал аналагично функцию ***mann_whitney_func***.
4. **Чистка неверно заполненных точек.**
- Создал пустой список. Запустил цикл по всем торговым точкам и добавил к этому списку все торговые точки, в которых не было заплачено ни одного рубля ни одним клиентом.
- Создал еще один пустой список. Запустил цикл по всем торговым точкам и добавил к этому списку все торговые точки, в которых пустует или контрольная, или тестовая группа.
5. **Расчет общих результатов АБ Теста.**
- Отбросил все торговые точки, которые были обнаружены в пункте 4.
- Изобразил гистограмму платежей, на которой различными цветами изображены группы “тест” и “контроль”.
- Применил функцию ***test_calc*** и сделал вывод (сравниваем средних платежей).
- Применил функцию ***test_calc*** и сделал вывод (сравниваем конверсию в платеж, то есть r1 и r2 - это ряды нулей и единиц, в зависимости от того, оплатил ли что-то клиент или нет).
- Применил функцию ***mann_whitney_func*** и сделал вывод.
6. **Сегментация результатов АБ Теста.**
- Повторил аналитику из пункта 5 в отношении Москвы. 
- Повторил аналитику из пункта 5 в отношении Санкт-Петербурга. 
- Запустил цикл по оставшимся городам. Провел такой же анализ.
7. **Отчет по АБ Тесту.**
- Создал пустой датафрейм. Запустил цикл по всем городам и торговым точкам.
- Заполнил для каждой торговой точки все поля, указанные в требовании к таблице.
- Выгрузил полученные результаты в Excel. Разнесите результаты по трем разным листам в зависимости от лейбла.

### Выводы (итоги):

1. **Таблица хранящая результаты A/B теста:**
- city - город
- id_trading_point  - уникальный идентификатор торговой точки, к которой прикреплен данный пользователь
- count_test - кол-во наблюдений в тестовой группе
- count_control - кол-во наблюдений в контрольной группе
- count_all - суммарное кол-во наблюдений
- percent_count - процент кол-ва наблюдений данной ТТ от всех наблюдений
- avg_payment_test - средний платеж в тестовой группе
- avg_payment_control - средний платеж в контрольной группе
- diff - разница между средними платежами
- sigma_test - стандартное отклонение платежей в тестовой группе
- sigma_control - стандартное отклонение платежей в контрольной группе
- ttest - значение статистического критерия (критерий Стьюдента для сравнения средних платежей)
- pvalue_ttest - pvalue статистического критерия (критерий Стьюдента для сравнения средних платежей)
2. **3 калькулятора в Excel**
- Калькулятор который считает какая выгода может быть получена от замены механики A на B при условии, что ей воспользуются N клиентов.
- Калькулятор который считает какая сумма может быть потеряна, если мы заменим механику А на механику В.
- Калькулятор который считает для каждой торговой точки кол-во наблюдений, необходимое для обнаружения разницы масштаба MDE (рассчитывать на основании стандартного отклонения платежей).
